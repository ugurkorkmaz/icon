Contributing to meticon
==============================

Thank you for considering contributing to the meticon project! Here are a few guidelines to help you get started.

Code of Conduct
---------------

Please note that this project is released with a [Contributor Code of Conduct](https://github.com/ugurkorkmaz/meticon/CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.

How to Contribute
-----------------

There are several ways you can contribute to this project:

-   Report a bug
-   Suggest a new feature
-   Submit a pull request

### Reporting a Bug

If you encounter a bug, please create an issue on the [issue tracker]([https://github.com/ugurkorkmaz/meticon/issues) and provide as much detail as possible, including steps to reproduce the issue and any relevant error messages.

### Suggesting a New Feature

If you have an idea for a new feature, please create an issue on the [issue tracker](https://github.com/ugurkorkmaz/meticon/issues) and provide as much detail as possible, including why you think the feature would be useful and any potential implementation details.

### Submitting a Pull Request

If you would like to contribute code to the project, please follow these steps:

1.  Fork the repository and create a new branch for your changes.
2.  Make your changes and commit them with clear commit messages.
3.  Push your changes to your forked repository.
4.  Submit a pull request to the [main repository](https://github.com/ugurkorkmaz/meticon/pulls) with a description of your changes.

Please ensure that your code follows the project's coding standards and that any new features are covered by unit tests. All pull requests will be reviewed by project maintainers, and may require additional changes before they can be merged.
